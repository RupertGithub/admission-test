1. What is Git and why is it used?

Github 是一個可以託管程式碼的服務平台，使用者可以將程式碼上傳到自己的 repository，將其設為開源或者是私有，也可以限制存取權限於特定團體，且能夠和合作的使用者們共同編輯一個專案，並提供
許多協助版本控制的特色例如:branch, fork, tag等等，增加協作的安全性和效率，因為這些特色讓Github成為眾多使用者使用的平台。

2. What is the difference between List, Dictionary, Tuple and Set in Python?

------------------------------------------------------------------------------------------------------------------------------
List -- 具有序列索引資料群結構，可寫作如下
list_ex = ['x','y','z'] #-- 中括號
list_ex[0] #is equal to 'x'
list_ex[1] #is equal to 'y'
list_ex[2] #is equal to 'z'
------------------------------------------------------------------------------------------------------------------------------
Tuple -- 具有序列索引資料群結構，和List的差別在於Tuple不能夠被改寫，可寫作如下
tuple_ex = ('x','y','z') #-- 小括號
tuple_ex[0] #is equal to 'x'
tuple_ex[1] #is equal to 'y'
tuple_ex[2] #is equal to 'z'
假設想要修改tuple的值是不行的
tuple_ex[2] = 'a'
會出現TypeError: 'tuple' object does not support item assignment
------------------------------------------------------------------------------------------------------------------------------
Dictionry -- 具有鍵值對的資料群結構，可寫作如下
dict_ex = {'name':'John', 'age':20, 'job':'engineer'} #-- 大括號
dict_ex['name'] #is equal to 'John'
dict_ex['age'] #is equal to 20
dict_ex['job'] #is equal to 'engineer'
------------------------------------------------------------------------------------------------------------------------------
set -- 如同數學意義的集合，資料在意義上是無序的，且每一筆資料都是獨一無二的，其中一種創造集合的方法是將List的資料轉入一個集合中
list_ex = ['1','1','34','qrq3']
set_ex = set(list_ex) #-- 將list_ex資料轉入set_ex
print(sorted(set_ex)) #-- set意義上是無序的，在print函數列印時會自動排序，或者是使用sorted函數來做排序
output: ['1', '34', 'qrq3']#-- 每一筆資料在集合中只會出現一次，因此即便list的重複資料(字串'1')轉入set就成為唯一一筆資料
------------------------------------------------------------------------------------------------------------------------------
